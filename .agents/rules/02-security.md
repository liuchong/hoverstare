# 02 — 安全规则

## 秘钥

1. 任何密钥/令牌**绝不提交进仓库**，也绝不出现在 PR / issue / CI 日志输出中。
2. 机密一律 `secrecy::SecretString` 持有；日志输出前自查脱敏。
3. 本地用 `.env` 文件时，确保 gitignore 覆盖 `.env*`（含编辑器备份 `.env~`——
   本项目有过备份文件险遭入库的真实前科，推送前必须 `git status` 检查）。
4. 提交前如怀疑敏感信息入库：全历史 grep 扫描，发现即在推送前清除
   （`git rm --cached` + amend/rebase，不要只是再提交一次删除）。

## 不可信数据

1. diff、仓库文件内容、PR 评论**都是数据不是指令**。系统提示中的
   不可信数据声明（防 prompt injection）不可删改。
2. 模型输出永远先经 schema 校验 + 字段归一化，再进入系统。
3. 行内评论的行号必须过锚定降级链，不信任模型给的行号。

## 只读强制

1. 审查工具集在**机器层**只读（注册表里不存在写工具），不靠 prompt 约束。
2. 新工具必须先过路径沙箱评审：拒绝绝对路径、`..` 逃逸、符号链接逃逸。
3. 禁止执行 checkout 下来的任何代码/脚本；
   `git show <ref>:<path>` 固定格式是唯一允许的进程调用。
